### Функциональные требования для демо-репозитория мини-приложения Telegram

#### Общие требования:
1. **Технологический стек**:
   - Frontend: React (с использованием функциональных компонентов и хуков).
   - Backend: (опционально, если требуется для демонстрации) Node.js/Express или mock-сервер для эмуляции API.
   - Библиотеки: React Router, Axios (или Fetch API), Redux Toolkit (или Context API для управления состоянием), Material-UI/Ant Design/Chakra UI (для UI-компонентов).

2. **Цель**:
   - Демонстрация функционала мини-приложения Telegram для регистрации чеков в программе лояльности производителя бытовой химии.
   - Минимальный, но функциональный интерфейс для взаимодействия с пользователем.

3. **Структура репозитория**:
   - Четкая структура папок: `src/components`, `src/pages`, `src/services`, `src/store`, `src/assets`.
   - Документация: `README.md` с описанием проекта, инструкцией по запуску и списком функционала.
   - Пример данных для тестирования (mock-данные).

---

#### Функциональные требования:

##### 1. **Авторизация через Telegram**
   - Пользователь должен авторизоваться через Telegram (используя Telegram Web Apps API).
   - После авторизации пользователь видит приветственное сообщение с именем и аватаром, полученными из Telegram.
   - Данные авторизации должны сохраняться в локальном состоянии (например, в Redux или Context API).

##### 2. **Загрузка чеков**
   - Пользователь может загрузить чек (в виде изображения или PDF).
   - При загрузке чека:
     - Отображается прогресс загрузки.
     - Выполняется валидация формата файла (например, только `.jpg`, `.png`, `.pdf`).
     - После успешной загрузки чек сохраняется в истории чеков.
   - Для демонстрации можно использовать mock API для эмуляции загрузки.

##### 3. **Просмотр истории чеков**
   - Пользователь может просмотреть список ранее загруженных чеков.
   - Для каждого чека отображается:
     - Дата загрузки.
     - Статус обработки (например, "Обрабатывается", "Принят", "Отклонен").
     - Превью изображения чека.
   - Возможность фильтрации чеков по статусу.

##### 4. **Новостной функционал**
   - На главной странице отображается блок с новостями от производителя.
   - Каждая новость включает:
     - Заголовок.
     - Краткое описание.
     - Ссылку на полную новость (открывается в новом окне).
   - Новости можно загружать из mock API или статического JSON-файла.

##### 5. **Интерфейс пользователя**
   - Простой и интуитивно понятный интерфейс.
   - Навигация между страницами:
     - Главная страница (новости).
     - Страница загрузки чека.
     - Страница истории чеков.
   - Использование адаптивного дизайна для корректного отображения на мобильных устройствах.

##### 6. **Обработка ошибок**
   - Уведомления об ошибках (например, при неудачной загрузке чека или проблемах с авторизацией).
   - Сообщения об успешных действиях (например, "Чек успешно загружен").

##### 7. **Дополнительные требования**
   - Локализация интерфейса (например, поддержка русского и английского языков).
   - Минимальная анимация для улучшения пользовательского опыта (например, при загрузке чека или переключении страниц).

---

#### Нефункциональные требования:
1. **Кроссбраузерность**: Приложение должно корректно работать в последних версиях Chrome, Firefox, Safari.
2. **Адаптивность**: Приложение должно быть оптимизировано для мобильных устройств.
3. **Код**:
   - Чистый и читаемый код с комментариями.
   - Использование ESLint и Prettier для поддержания единого стиля кода.
4. **Документация**:
   - Подробное описание проекта в `README.md`.
   - Инструкция по запуску проекта локально.
   - Описание используемых технологий и их назначения.

```

│   ├── index.js         # Точка входа
├── .eslintrc.js         # Конфигурация ESLint
├── .prettierrc          # Конфигурация Prettier
├── package.json         # Зависимости проекта
├── README.md            # Документация
```

#### Пример сценария использования:
1. Пользователь открывает мини-приложение Telegram.
2. Авторизуется через Telegram.
3. На главной странице видит новости от производителя.
4. Переходит на страницу загрузки чека, загружает чек.
5. Переходит на страницу истории чеков, видит загруженные чеки и их статусы.
